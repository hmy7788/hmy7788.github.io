## 1. UML (Unified Modeling Language)

### 1) UML 개요
* 원활한 의사소통을 위한 **표준화된 범용 모델링 언어**
* 객체지향 방법론(Rumbaugh, Booch, Jacobson)의 장점을 통합
* 사물, 관계, 다이어그램으로 구성됨

---

### 2) UML 구성 요소

#### 사물 (Things)

| 종류 | 정의 |
| :--- | :--- |
| **구조 사물** | 개념적, 물리적 요소 (클래스, 인터페이스, 유스케이스, 노드 등) |
| **행동 사물** | 각 요소들의 행위, 상호작용 |
| **그룹 사물** | UML 요소들을 묶어서 그룹화 |
| **주해 사물** | UML에 대한 부가적 설명, 주석 |

#### 관계 (Relationships)
* 사물과 사물의 연관성을 표현
* **종류**: 연관, 집합, 포함, 일반화, 의존, 실체화 관계

#### 다이어그램 (Diagrams)
* 사물과 관계를 정형화된 도형으로 표현
* **6개의 구조적(정적) 다이어그램**과 **7개의 행위(동적) 다이어그램**으로 분류

#### 스테레오 타입 (Stereotype)
* 기본 요소 이외의 새로운 요소를 표현하기 위한 확장 구조
* `<< >>` 기호를 사용하여 표기

| 기호 | 정의 |
| :--- | :--- |
| `<<include>>` | 유스케이스의 필수 포함 관계 |
| `<<extend>>` | 유스케이스의 선택적 확장 관계 |
| `<<interface>>` | 추상 메소드로만 이루어진 클래스 |
| `<<entity>>` | 기억 장치에 저장될 정보 표현 |
| `<<boundary>>` | 시스템 외부와의 상호작용 담당 |
| `<<control>>` | 시스템 기능, 제어 담당 |

---

## 2. 관계 (Relationship) 종류

### 1) 연관 관계 (Association)
* 둘 이상의 사물이 서로 관련되어 있음 (지속적 참조)
* **표기법**: 실선 화살표 (양방향일 경우 화살표 없는 실선으로만 표현)
* 다중도를 사물 양 끝단에 표기
  * `*` : 없거나 다수
  * `n` : n개
  * `n..m` : 최소 n개, 최대 m개
  * `n..*` : n개 이상

![연관 관계 예시](<../imgs/UML 연관관계.png>)

### 2) 집합 관계 (Aggregation)
* 사물이 다른 사물에 포함되어 있는 관계 (두 사물은 서로 **독립적**)
* **표기법**: 전체 사물 쪽에 **속이 빈 마름모**를 붙인 실선

![집합 관계 예시](<../imgs/UML 집합관계.png>)

### 3) 포함 관계 (Composition)
* 사물이 다른 사물에 포함되어 있는 관계 (두 사물은 서로 **종속적**)
* **표기법**: 전체 사물 쪽에 **속이 찬 마름모**를 붙인 실선

![포함 관계 예시](<../imgs/UML 포함관계.png>)

### 4) 일반화 관계 (Generalization)
* 상위 사물(일반적인 속성)과 하위 사물(구체적인 속성) 간의 관계
* **표기법**: 상위 사물 쪽에 **속이 빈 삼각 화살표**를 붙인 실선

![일반화 관계 예시](<../imgs/UML 일반화관계.png>)

### 5) 의존 관계 (Dependency)
* 필요에 의해 짧은 시간 동안만 관계를 유지하는 형태 (사물의 변화가 다른 사물에 영향을 미침)
* **표기법**: 영향을 받는 사물 쪽에 **점선 화살표** 표현

![의존 관계 예시](<../imgs/UML 의존관계.png>)

### 6) 실체화 관계 (Realization)
* 공통적인 기능을 상위 사물로 그룹화하는 관계
* **표기법**: 상위 사물 쪽에 **속이 빈 삼각 화살표**를 붙인 **점선**

![실체화 관계 예시](<../imgs/UML 실체화관계.png>)

### 1) 구조적 다이어그램 상세

#### 클래스 다이어그램 (Class)
* 클래스 간 구조적 관계 표현
* 구성 요소: 클래스명, 속성, 연산, 접근 제어자

> **접근 제어자 표기법**
> * `-` (private): 클래스 내부에서의 접근만 허용
> * `+` (public): 클래스 외부에서의 접근도 허용
> * `#` (protected): 동일 패키지 및 상속받은 자식 클래스에서의 접근만 허용
> * `~` (default): 동일 패키지 클래스에서의 접근만 허용
{: .prompt-warning }

#### 객체 다이어그램 (Object)
* 클래스의 인스턴스를 객체 간의 관계로 표현
* 특정 시점의 시스템 구조 파악

#### 컴포넌트 다이어그램 (Component)
* 구현 단계에서 사용
* 컴포넌트 간 구성 및 연결 상태 표현

#### 배치 다이어그램 (Deployment)
* 구현 단계에서 사용
* 물리적 아키텍처 표현 (서버, 노드, 컴포넌트 등 하드웨어 배치)

#### 복합체 구조 다이어그램 (Composite Structure)
* 복합적 구조를 가진 컴포넌트나 클래스의 내부 구조 표현

#### 패키지 다이어그램 (Package)
* 시스템의 서로 다른 패키지 간 의존 관계 표현

---

### 2) 행위적 다이어그램 상세

#### 유스케이스 다이어그램 (Use Case)
* 기능 중심 모델링 결과물을 사용자 관점에서 표현

#### 상태 다이어그램 (State)
* 특정 기간, 특정 객체를 대상으로 이벤트 활성에 따른 상태 변화를 표현

#### 활동 다이어그램 (Activity)
* 객체의 프로세스나 로직 처리 흐름을 순서에 따라 표현 (순서도와 유사)

#### 순차/절차 다이어그램 (Sequence)
* 객체의 생성, 소멸, 객체 간 메시지 등 동적인 면을 시간에 따라 표현

| 구성요소 | 정의 |
| :--- | :--- |
| **생명선** | - 객체에서 아래로 뻗어나가는 점선<br>- 시간의 흐름에 따라 길어짐 |
| **실행** | - 생명선 위의 직사각형<br>- 해당 객체가 활성화 상태임을 의미 |
| **메시지** | - 객체 간 이동하는 화살표<br>- 요청과 응답을 통해 주고받는 데이터 |

#### 통신 다이어그램 (Communication)
* 순차 다이어그램의 요소에 더해 객체들 간의 연관 관계까지 표현

#### 타이밍 다이어그램 (Timing)
* 객체의 상태 변화와 시간에 따른 상호작용을 타이밍 제약 조건과 함께 표현

#### 상호작용 개요 다이어그램 (Interaction Overview)
* 여러 다이어그램 사이의 제어 흐름을 표현
