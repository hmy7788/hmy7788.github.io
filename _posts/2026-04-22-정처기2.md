## 1. UI 설계
### 1) UI 설계 개요
#### UI 설계 정의
* 정의된 UI 요구사항, 표준 및 지침 바탕 -> UI 설계 단계

#### UI 시스템 필수 기능
* 사용자 입출력 지원 -> 프롬프트 기능
* 전달되는 명령 올바른지? -> 검증 기능
* 에러의 처리, 에러에 대한 메세지 처리 기능
* 도움말 기능

#### UI 설계 원칙

|원칙|정의|
|:---|:---|
|**직관성**|즉시 사용하는 정도|
|**유효성**|목적을 정확하게 달성하는 정도|
|**학습성**|쉽게 배우고 익힐 수 있는 정도|
|**유연성**|요구사항 수용, 오류 최소화하는 정도|

> 비슷한 의미의 다른 용어 출제 가능

#### UI 설계 지침
* 오류 최소화, 적은 노력으로 구현
* 기능에 대한 구체적인 방법 제시

---

### 2) UI 설계 절차

|절차|정의|
|:---|:---|
|문제 정의|문제를 자유롭게 기술|
|사용자 모델링|사용자의 SW 기술 -> 초급자, 중급자, 숙련자 그룹으로 분류 <br>사용자에 따라 전략 세움|
|작업 분석|문제를 세분화, 구체화|
|컴퓨터 장치 및 기능 정의|어떤 컴퓨터 장치를 통해 수행하는지 정의 <br>일반적인 컴퓨터 기능(복사, 붙여넣기 등) 제공|
|사용자 인터페이스 정의|상호작용 장치 식별 <br>시스템 상태 인식|
