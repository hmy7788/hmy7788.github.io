## 1. UI 요구사항 확인

### 1) UI (User Interface)
#### UI 정의 및 특징
* 사용자와 컴퓨터 상호 간 원활한 소통 도와주는 연계 시스템
* 단순한 상호작용, 실행 오류를 줄이기 위해 UI 발전
* 단순한 기능 전달 X -> 의미를 전달 O

#### UI 종류

|종류|정의|
|:---|:---|
|**CLI**(Command Line Interface)|명령 문장열 -> 시스템과 상호작용|
|**GUI**(Graph User Interface)|그래픽 요소 -> 시스템과 상호작용|
|**NUI**(Natural User Interface)|사람 음성, 촉각 -> 시스템과 상호작용|

---

### 2) UI 표준과 지침

#### UI 표준과 지침 정의
* **UI 표준**: 모든 UI에 공통적으로 적용되어야 하는 내용
* **UI 지침**: UI 개발 시 지켜야 할 공통 세부 개발 방향

#### 레이아웃 구성 요소

| 구성요소 | 정의 |
| :--- | :--- |
| **Indicator** | 상태 알림 영역 |
| **Header** | 정체성(로고, 사이트) 영역 |
| **Navigator** | 현재 서비스 위치, 다른 서비스 이동 지원 영역 |
| **Content** | 전달되는 정보 영역 |
| **Button** | 별도 요소 표현 영역 |
| **Footer** | 정보 제공자 정보 영역 |

#### UI element 종류

| 종류 | 정의 |
| :--- | :--- |
| **텍스트 박스** | 정보 입력 공간 |
| **라디오 박스** | 다수 나열 항목 -> **하나**의 값 선택 |
| **토글 버튼** | 하나의 버튼 -> **두 상태(On/Off)** 번갈아가며 설정 |
| **콤보 박스** | 드롭 다운 리스트 -> 원하는 항목 하나 선택 |
| **체크 박**스 | 다수 나열 항목 -> **하나 이상** 값 선택 |
| **명령 버튼** | 지정된 명령 수행 |

---

### 3) 한국형 웹 콘텐츠 접근성 지침 (KWCAG)
#### 웹 사이트 개발 시 고려사항

|고려사항|정의|
|:---|:---|
|**웹 표준**|다양한 브라우저 -> 동일 구현|
|**웹 접근성**|다른 환경(ex. 장애인, 고령자) -> 동일 정보 제공|
|**웹 호환성**|다른 시스템 환경 (ex. 모바일, 맥) -> 동일 서비스 제공| 

#### 웹 콘텐츠 접근성 정의
* 디지털 약자(장애인, 고령자)가 웹 사이트에서 제공하는 정보에 비장애인과 동등하게 접근하고 이해할 수 잇도록 보장하는 지침
* 대체 형식의 콘텐츠 함께 제공

## 2. UI 프로토타입
### 1) UI 프로토타입 개요
#### UI 프로토타입 정의 및 특징
* UI를 최대한 단순하게 제작한 시제품
* 필요 요소만 포함 -> 보완, 개선
* **장점**: 사용자 설득, 사전 결함 발견
* **단점**: 비용 증가, 중요 기능 누락 가능성

### 2) UI 프로토타입 전략
#### Paper Prototype
* 손으로 직접 스케치 -> 프로토타입 제시
* **장점**: 사전 지식 없이 수행 가능, 변경사항 즉시 반영 가능
* **단점**: 실제 테스트 불가, 복잡한 UI 요소 간 상호 관계 반영 어려움

#### Digital Prototype
* 컴퓨터 소프트웨어 -> 프로토타입 제시
* **장점**: 실제 테스트 가능, 수정 용이
* **단점**: 숙력된 전문가 필요, 사용법 숙지에 시간 필요
